التحليل الهيكلي لخوارزميات البرمجة التنافسية

مسألة: Frog 1 (DP Top-Down)
هذا المستند يقسم كود البرمجة الديناميكية (DP) إلى أجزائه الرئيسية. عند الضغط على أمثلة مشابهة، سيتغير الكود البرمجي أمامك ليوضح كيف تختلف كتابة هذا الجزء باختلاف المسألة.
1

المتغيرات وحالة المسألة (State)

#include <iostream>
using namespace std;

int a[200000];
int dp[200000];
كود معدل: مسألة حقيبة الظهر (Knapsack)
#include <iostream>
using namespace std;

int weight[105], value[105];
int dp[105][100005]; // [item][weight]
المصفوفة dp تستخدم لتخزين الإجابات التي تم حسابها مسبقاً (Memoization Array) لتجنب إعادة الحساب. المصفوفة a لتخزين المدخلات (ارتفاعات الأحجار). حجمها 200000 بناءً على قيود المسألة N ≤ 10^5.
تغير المعطيات
في مسألة مثل حقيبة الظهر (Knapsack 0/1)، الحالة تعتمد على عاملين: (رقم العنصر، والوزن المتبقي). لذلك ستتغير مصفوفة الـ DP من بُعد واحد إلى بُعدين كما يظهر في الكود، ليصبح لدينا مصفوفة ثنائية الأبعاد.
2

الحالات الأساسية (Base Cases)

int f(int i) {
    // الحالات الأساسية
    if (i == 0) {
        return 0;
    }

    if (i == 1) {
        return abs(a[1] - a[0]);
    }
كود معدل: مسألة المسارات في شبكة (Grid Paths)
int f(int r, int c) {
    // 1. حالة الخروج عن حدود الشبكة أو الاصطدام بحاجز
    if (r < 0 || c < 0) {
        return 0; // صفر طرق ممكنة
    }
    
    // 2. حالة الوصول لنقطة البداية (الهدف)
    if (r == 0 && c == 0) {
        return 1; // وجدنا طريقاً واحداً
    }
هذه شروط التوقف التي تمنع الدالة من الاستدعاء الذاتي إلى المالانهاية.
- التكلفة للوصول للحجر 0 هي 0.
- التكلفة للوصول للحجر 1 هي حصراً القفز من الحجر 0 (لأنه لا يوجد حجر قبله).
تغير المعطيات
في مسائل البحث في الشبكات ثنائية الأبعاد (Grid Paths)، ستتغير شروط التوقف لتعكس الإحداثيات كما يظهر في الكود المقابل. نتحقق أولاً من عدم الخروج خارج الشبكة، ثم نتحقق من الوصول للهدف لإرجاع 1 كطريقة ناجحة.
3

تقنية الحفظ (Memoization)

    // إذا تم حساب الحل من قبل
    if (dp[i] != -1) {
        return dp[i];
    }
كود معدل: مسائل القيم السالبة المتوقعة
    // لا يمكن استخدام -1 إذا كانت النواتج قد تكون سالبة
    // نستخدم قيمة مستحيلة مثل سالب مليار (1e9-)
    if (dp[i] != -1e9) {
        return dp[i];
    }
هذا السطر هو جوهر الـ DP! نتحقق: هل قمنا بحساب الإجابة لهذا الـ i سابقاً؟ إذا كانت القيمة لا تساوي -1 فهذا يعني "نعم"، فنقوم بإرجاعها فوراً بدلاً من تفرع الاستدعاءات. هذا يقلل التعقيد الزمني بشكل هائل.
تغير المعطيات
إذا كانت المسألة (مثل مسائل الـ Max Profit المعقدة) تسمح بإجابات تكون قيمتها النهائية -1، فلا يمكننا استخدامها كعلامة للتهيئة! سنضطر لتغيير الشرط في الكود لاستخدام قيمة مستحيلة جداً مثل -1e9.
4

القرار / معادلة الانتقال (Transition)

    // حساب وتخزين الناتج (لخيارين فقط)
    dp[i] = min(
        f(i - 1) + abs(a[i] - a[i - 1]),
        f(i - 2) + abs(a[i] - a[i - 2])
    );

    return dp[i];
}
كود معدل: مسألة (Frog 2) تسمح بـ K قفزات
    // استبدال الخيارين الثابتين بحلقة تكرار لـ K من الخيارات
    int res = 1e9; // تهيئة مبدئية بقيمة كبيرة جداً
    
    for (int j = 1; j <= K; j++) {
        if (i - j >= 0) {
            res = min(res, f(i - j) + abs(a[i] - a[i - j]));
        }
    }
    
    dp[i] = res;
    return dp[i];
}
هنا القرار الرياضي: للوصول للحجر i بأقل تكلفة، نقارن بين خيارين ثابتين فقط:
1. القفز من الحجر السابق i-1.
2. القفز من الحجر الذي قبله بخطوتين i-2.
نأخذ الحد الأدنى min، ونحفظه.
تغير المعطيات
في مسألة Frog 2، يُسمح بالقفز من 1 إلى K خطوات. هنا الدالة min البسيطة لن تكفي. يجب تغيير الكود واستخدام حلقة تكرار (For loop) كما يظهر في الكود المجاور للمرور على كل الـ K قفزات الممكنة واختيار الأفضل منها.
5

التهيئة والدالة الرئيسية (Main)

int main() {
    // تهيئة المصفوفة بـ -1 مرة واحدة فقط
    for (int i = 0; i < 200000; i++) {
        dp[i] = -1;
    }

    int n;
    cin >> n;

    for (int i = 0; i < n; i++) {
        cin >> a[i];
    }

    cout << f(n - 1) << "\n";
}
كود معدل: مسألة تتضمن اختبارات متعددة (Test Cases)
int main() {
    int t; cin >> t; // قراءة عدد الاختبارات
    
    while (t--) {
        int n; cin >> n;
        
        // التهيئة يجب أن تتم بالداخل قبل كل اختبار!
        for (int i = 0; i <= n; i++) {
            dp[i] = -1;
        }
        
        // ... قراءة المصفوفة وطباعة الناتج ...
    }
}
في البداية، يجب ملء مصفوفة dp بالقيمة -1 للدلالة على عدم وجود أي قيمة محسوبة مسبقاً. ثم نقرأ المدخلات N ومصفوفة الارتفاعات، وأخيراً نطبع نتيجة استدعاء الدالة.
تغير المعطيات
إذا كانت المسألة تحتوي على اختبارات متعددة (Multiple Test Cases - T)، فمن الخطأ الفادح تهيئة مصفوفة dp مرة واحدة فقط. يجب أن يتغير الكود لتنتقل حلقة التهيئة لتكون داخل حلقة الـ while(t--) ليتم تصفير الذاكرة قبل كل اختبار جديد.
