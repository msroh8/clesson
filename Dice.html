تغير المعطيات

الخيارات والتحسين (Inner Loop & Micro-Optimization)

        for (int k = 1; k <= 6; k++) {
            if (x - k >= 0) {
                ways += dp[x - k];
                
                // small optimization: Subtraction is faster than % MOD
                if (ways >= MOD) ways -= MOD; 
            }
        }
كود معدل: لو كانت الخيارات مصفوفة عملات (Coin Change)
        // الخيارات ليست الأرقام من 1 لـ 6، بل مصفوفة ديناميكية (coins)
        for (int coin : coins) {
            if (x - coin >= 0) {
                // حساب عدد الطرق (Coin Change - Combinations)
                ways += dp[x - coin];
                if (ways >= MOD) ways -= MOD; 
            }
        }
هنا نستكشف الخيارات الممكنة (رمي النرد يعطي رقماً من 1 إلى 6). للوصول للهدف x، نضيف جميع طرق الوصول للأهداف السابقة x - k.
نقطة احترافية: بدلاً من كتابة ways %= MOD المكلفة في الزمن، الكود يستخدم عملية الطرح -= وهي أسرع بكثير على المعالج (Micro-optimization).
تغير المعطيات
في مسألة النرد الكلاسيكية خياراتك ثابتة (1 إلى 6). لكن لو كانت المسألة هي Coin Change (حيث لديك فئات معينة من العملات مثل 2، 5، 10)، فإن الحلقة الداخلية تتغير لتمر على مصفوفة العملات المُدخلة coins بدلاً من المرور على أرقام ثابتة.

بعد انتهاء الحلقة الداخلية من جمع كل الطرق للوصول للرقم x، نقوم بحفظ الناتج في المصفوفة dp[x]. بما أننا تأكدنا طوال الوقت أن الرقم لا يتجاوز الـ MOD، فمن الآمن عمل Cast إلى int لتوفير المساحة. وفي النهاية نطبع الخلية التي تمثل الهدف النهائي dp[n].
تغير المعطيات
في مسألتنا (النرد)، نحن مهتمون بقيمة محددة وهي المجموع n ولذلك الحل هو حصراً dp[n]. لكن في مسائل أخرى مثل أطول سلسلة متزايدة (LIS)، الإجابة (أطول سلسلة) قد تنتهي عند أي عنصر في المصفوفة، وليس بالضرورة العنصر الأخير. لذلك بدلاً من طباعة dp[n]، يجب عمل حلقة إضافية للمرور على المصفوفة واستخراج القيمة العظمى Max.
