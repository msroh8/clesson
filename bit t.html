حيل التعامل مع البِتّات

Bit Manipulation Tricks — شرح تفاعلي مع أمثلة حيّة ومحاكاة بصرية

المقدّمة والافتراضات

سنفترض وجود المتغيرين التاليين طوال الدرس:

int x;       // some non-negative integer
int k;       // 0-based bit index (0 = least significant bit)

وسنستخدم أقنعة (masks) مثل القناع أدناه، والذي يكون فيه البِت رقم k فقط مساويًا للقيمة 1.

1 << k
🎯 كيف تستخدم هذه الصفحة؟
  • اضغط على زر ⓘ بجوار اسم كل حيلة لقراءة: متى تستخدم؟ وما المشاكل الشهيرة التي تحلّها؟
  • انقر على أي سطر برمجي داخل خلفية الكود الداكنة لتظهر نافذة تشرح السطر كلمةً كلمة.
  • انقر على الرموز المظلّلة مثل العوامل البِتّية لفتح شرح مفصّل لكل رمز.
  • جرّب الحيلة بنفسك عبر تغيير القيم ثم الضغط على زر «نفّذ».

الحيل المغطّاة

١. فحص البِت رقم k ٢. ضبط البِت إلى 1 ٣. تصفير البِت ٤. قلب البِت ٥. إزالة أقل بِت معيّن ٦. فحص قوة الـ 2 ٧. عزل أقل بِت معيّن

١ فحص ما إذا كان البِت رقم k يساوي 1

Check if bit k is set

bool isSet = (x >> k) & 1;
كيف تعمل؟
  • الإزاحة لليمين تنقل البِت رقم k إلى أقصى اليمين.
  • ثم عملية AND مع 1 تُبقي البِت الأيمن فقط وتصفّر باقي البِتّات.
  • الناتج يكون 0 أو 1، وهو يعادل false أو true منطقيًا.

مثال: افحص ما إذا كان البِت رقم 1 في x = 10 مُعيَّنًا.

int x = 10;     // 10 = 1010 (in binary)
int k = 1;

bool isSet = (x >> 1) & 1; // 1010 >> 1 = 101;  101 & 1 = 1

وهنا دالة جاهزة تُعيد ما إذا كان البِت رقم k في n مساويًا لـ 1:

bool kthBitIsSet(int n, int k) {
    return (n >> k) && 1;
}
جرّب بنفسك

٢ ضبط البِت رقم k إلى 1

Set bit k to 1

x |= (1 << k);
الفكرة:
  • العامل | يعيّن البِت إلى 1 إذا كان على الأقل أحد الطرفين يحتوي 1.
  • بهذا نجبر البِت رقم k أن يصبح 1، بينما تبقى باقي البِتّات كما هي.

مثال:

int x = 4;      // 0100
x |= (1 << 0);  // 0100 | 0001 = 0101 (5)
جرّب بنفسك

٣ تصفير البِت رقم k (جعله 0)

Clear bit k

x &= ~(1 << k);
الفكرة:
  • نبدأ بقناع فيه البِت رقم k فقط مساوٍ لـ 1.
  • نعكس القناع باستخدام ~ ليصبح البِت رقم k هو 0 وباقي البِتّات 1.
  • عملية AND مع القناع المعكوس تصفّر البِت رقم k فقط وتُبقي الباقي.

مثال:

int x = 13;         // 1101
x &= ~(1 << 2);     // ~(0100) = ...1011
                    // 1101 & 1011 = 1001 (9)

الآن البِت رقم 2 أصبح 0.

جرّب بنفسك

٤ قلب (تبديل) البِت رقم k

Toggle bit k

x ^= (1 << k);
الفكرة:

العامل ^ (XOR) يقلب البِت عندما يكون القناع فيه 1:

  • الصفر المُقارن بواحد يصبح واحدًا.
  • الواحد المُقارن بواحد يصبح صفرًا.

مثال:

int x = 5;     // 0101
x ^= (1 << 2); // 0101 ^ 0100 = 0001 (1)

البِت رقم 2 كان 1 ← أصبح الآن 0.

جرّب بنفسك

٥ إزالة أقل بِت معيّن

Remove the lowest set bit

هذه حيلة مشهورة جدًّا:

x = x & (x - 1);

هذا السطر يزيل (يُصفّر) أقل بِت معيّن (أول 1 من جهة اليمين) في x.

لماذا تعمل هذه الحيلة؟

فكّر ماذا يحدث للبِتّات عندما نطرح 1:

  • عند تنفيذ x - 1، نمشي من اليمين حتى نجد أول بِت قيمته 1.
  • هذا البِت الأقل (أقرب 1 لليمين) يتحوّل إلى 0.
  • كل البِتّات التي على يمينه (والتي كانت 0) تتحوّل إلى 1 بسبب الاقتراض في الطرح.

إذًا حول أقل بِت معيّن، النمط يكون بهذا الشكل:

x:       .... 1 0 0 0 0
x - 1:   .... 0 1 1 1 1

عند إجراء AND:

x        = .... 1 0 0 0 0
x - 1    = .... 0 1 1 1 1
-------------------------
x&(x-1)  = .... 0 0 0 0 0
  • أقل بِت معيّن يصبح 0 بسبب حاصل ضرب بِتّي 1 & 0.
  • كل البِتّات التي على يمينه كانت 0 في x الأصلية، وبالتالي ناتج الـ AND هو 0.
  • كل البِتّات التي على يساره لا تتأثر، لأن الطرح لم يغيّرها، فتبقى كما هي بعد AND.

لذلك x & (x - 1) تساوي قيمة x نفسها لكن مع أقل بِت معيّن مصفّرًا.

مثال:

x           = 12 = 1100
x - 1       = 11 = 1011
x & (x - 1) =  8 = 1000

إذًا قمنا بإزالة أقل بِت معيّن (البِت رقم 2).

٦ فحص ما إذا كان x قوة للعدد 2

Check if x is a power of 2

سنفترض أن x عدد موجب، ونستخدم الحيلة السابقة:

bool isPowerOfTwo = (x & (x - 1)) == 0;
لماذا تعمل هذه الحيلة؟
  • العدد الذي هو قوة للعدد 2 يحتوي في الثنائي على بِت واحد فقط مُعيَّن.
  • عند تنفيذ x & (x - 1) نقوم بإزالة أقل بِت معيّن.
  • إذا كان هناك بِت واحد فقط معيّن، فبعد إزالته تصبح القيمة 0.

إذًا النتيجة تكون صفرًا إذا وفقط إذا كان x يمتلك بِتًا واحدًا معيّنًا، أي أنه قوة للعدد 2.

أمثلة:

x = 8  (1000)  ->  x & (x-1) = 1000 & 0111 = 0000   power of 2
x = 12 (1100)  ->  x & (x-1) = 1100 & 1011 = 1000   NOT power of 2
جرّب بنفسك

٧ الحصول على أقل بِت معيّن فقط

Isolate the lowest set bit

int lowest = x & -x;

هنا -x هو المتمّم الثنائي (two's complement) للعدد x.

الفكرة:

التعبير x & -x يعزل أقل بِت معيّن (أول 1 من جهة اليمين) ويصفّر جميع البِتّات الأخرى.

مثال:

x                     = 12 = ...00001100
-x (two's complement) =      ...11110100
x & -x                =      ...00000100 = 4

أقل بِت معيّن في x يمثّل القيمة 4 (أي البِت رقم 2).
