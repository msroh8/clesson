في مسألة Minimizing Coins، الهدف يختلف. نحن لا نريد "كم طريقة للوصول للهدف"، بل "ما هو أقل عدد ممكن من العملات للوصول للهدف". لذلك لا نحتاج للـ MOD لأن الناتج سيكون رقماً صغيراً (أقل من target)، بل نُعرف ثابت INF ليمثل رقماً كبيراً جداً (المالانهاية).

نهيئ مصفوفة dp بحجم target + 1 لنصل إلى الخلية الأخيرة بنجاح. ونضع بداخل كل خلية 0 لأننا سنقوم بحساب عملية الجمع التراكمي.
الحالة الأساسية dp[0] = 1 تعني: "هناك طريقة واحدة صحيحة للحصول على المجموع صفر، وهي المجموعة الفارغة (عدم أخذ أي عملة)".
تغير المعطيات
في مسألة البحث عن الحد الأدنى Minimizing Coins، يتغير المعنى بالكامل. نحن نهيئ المصفوفة برقم المالانهاية INF لأننا سنبحث عن القيمة الأقل (min) (ولو وضعنا 0 فستكون الدالة min دائماً صِفراً وهو خطأ). وتصبح الحالة الأساسية dp[0] = 0 وتعني: عدد العملات اللازم للحصول على 0 هو 0 عملة.

معادلة الانتقال (Transition Equation)

            // هل يمكن استخدام هذه العملة؟
            if (w - c >= 0) {
                // عدد طرق الوزن (w) = مجموع طرق الأوزان (w-c)
                dp[w] = (dp[w] + dp[w - c]) % MOD;
            }
        }
    }
كود معدل: معادلة Minimizing Coins
            // هل يمكن استخدام هذه العملة؟
            if (w - c >= 0) {
                // أقل عدد عملات للوزن (w) = 
                // الأصغر بين القيمة الحالية و (الوزن السابق + 1 عملة)
                dp[w] = min(dp[w], dp[w - c] + 1);
            }
        }
    }
هذه هي نواة الـ DP. نتحقق أولاً أن العملة c لا تجعل الوزن سالباً w - c >= 0.
إذا كانت صالحة، فإننا نضيف عدد طرق الوصول للوزن السابق dp[w - c] إلى الخلية الحالية dp[w]. مع أخذ الـ MOD في كل خطوة لتجنب خطأ التجاوز الحسابي (Overflow).
تغير المعطيات
بالعودة لمسألة البحث عن الحد الأدنى للعملات Minimizing Coins، سنستبدل عملية الجمع بعملية التفضيل للحد الأدنى min. وسنقوم بإضافة + 1 للخلية السابقة لأننا "صرفنا" عملة واحدة إضافية الآن من نوع c للانتقال من dp[w - c] إلى dp[w].

كود معدل: التعامل مع الحالات المستحيلة (Unreachable States)
    // في مسائل الـ Minimization إذا بقيت الخلية بقيمتها الابتدائية INF
    // فهذا يعني أنه من المستحيل الوصول للمجموع المستهدف
    if (dp[target] == INF) {
        cout << -1 << '\n';
    } else {
        cout << dp[target] << '\n';
    }
    
    return 0;
}
بما أننا اعتمدنا أسلوب Bottom-Up في بناء الحل بدءاً من 1 وصولاً إلى target، فإن الإجابة النهائية ستكون ببساطة هي قيمة الخلية الأخيرة في المصفوفة dp[target]. وإذا كان من المستحيل الوصول للهدف، ستكون قيمتها تلقائياً الصفر الذي وضعناه في البداية.
تغير المعطيات
في مسألة Minimizing Coins، يُطلب عادةً طباعة -1 إذا كان من المستحيل تكوين المجموع باستخدام العملات المعطاة. لمعرفة ذلك برمجياً، نتحقق هل الخلية الأخيرة dp[target] لا تزال تحتفظ بقيمة INF التي وضعناها في البداية؟ إذا نعم، نطبع -1، وإلا نطبع الناتج الفعلي.
