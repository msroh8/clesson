التحليل الهيكلي لخوارزميات البرمجة التنافسية

مسألة: Vacation (2D State DP)
هذا المستند يحلل مسألة تعتمد على الحالة ثنائية الأبعاد (2D State DP). عند الضغط على أمثلة مشابهة، سيتغير الكود البرمجي أمامك ليوضح كيف تختلف كتابة هذا الجزء لجعله أكثر ديناميكية أو ليناسب مسألة مختلفة.
1

المتغيرات وحالة المسألة (State)

#include <iostream>
using namespace std;

int dp[200000][3];
int d[200000][3];
كود معدل: تمثيل حالة معقدة (Bitmask DP)
#include <iostream>
using namespace std;

// في حالة Bitmask، البعد الأول يمثل المجموعات الفرعية كأرقام ثنائية
int dp[1 << 20][20]; // [mask][last_visited]
هنا تطورت حالة المسألة لتصبح ثنائية الأبعاد (2D Array). المصفوفة dp[i][j] تخزن أقصى عدد من النقاط يمكن جمعه حتى اليوم i بشرط أن يكون النشاط المختار في هذا اليوم هو j (حيث أن 0 و 1 و 2 تمثل الأنشطة الثلاثة).
تغير المعطيات
في مسائل متقدمة مثل مسألة البائع المتجول (TSP)، الحالة تعتمد على "مجموعة المدن المُزارة" و "المدينة الحالية". يتم استخدام تقنية الـ Bitmask DP حيث يُمثل البُعد الأول بالحجم 2^N لتخزين المجموعات كأرقام ثنائية.

int f (int i, int type) {
  
    if (i == 0) {

كود معدل: إضافة عوائق (Obstacles) للـ Base Cases
int f (int i, int type) {
    // إذا كان هذا النشاط محظوراً في هذا اليوم
    if (is_forbidden[i][type]) {
        return -1e9; // قيمة مستحيلة لأننا نبحث عن Max
    }
    
    if (i == 0) {
        return d[i][type];
    }
هذا هو شرط التوقف الذي يمنع الاستدعاء اللانهائي.
عندما نصل إلى اليوم الأول i == 0، فإن أقصى عدد من النقاط يمكن جمعه هو ببساطة قيمة النقاط المخصصة للنشاط type في ذلك اليوم.
تغير المعطيات
إذا تم تعديل المسألة بحيث تصبح بعض الأنشطة "مغلقة" أو "محظورة" في أيام معينة (عوائق)، يجب إضافة حالة أساسية إضافية قبل التحقق من اليوم 0، حيث نُرجع قيمة سالبة ضخمة (سالب ما لا نهاية) لتجنب اختيار هذا المسار في حسابات الـ Max.

تقنية الحفظ (Memoization)

    if (dp[i][type] != -1) {
        return dp[i][type];
    }
كود معدل: تمييز الحالات المحسوبة عن غير المحسوبة
    // يمكن استخدام مصفوفة منطقية إضافية (visited)
    // هذا أأمن إذا كانت إجابة المسألة قد تساوي 1-
    if (visited[i][type]) {
        return dp[i][type];
    }
    visited[i][type] = true; // نضعها كـ true بعد الحساب بالأسفل
عملية الحفظ المعتادة للبرمجة الديناميكية. إذا وجدنا أن القيمة في dp[i][type] لا تساوي القيمة الابتدائية -1، نعلم فوراً أننا قمنا بحساب هذا المسار مسبقاً، فنُرجع الناتج المحفوظ اختصاراً للوقت.
تغير المعطيات
كما ذكرنا، استخدام -1 يعمل جيداً إذا كانت النواتج المتوقعة دائماً موجبة كحالتنا. ولكن في مسألة يمكن أن تكون إجابتها الصافية -1، يُفضل المحترفون إنشاء مصفوفة bool visited[N][3] مستقلة تماماً لمعرفة هل تم حساب الحالة أم لا، بدلاً من الاعتماد على قيمة الـ dp نفسها.

معادلة الانتقال (Transition) وتطبيق الشروط

  if(type == 0){
    dp[i][type] = max(f(i - 1, 1), f(i - 1, 2)) + d[i][type];
  }
  else if(type == 1){
    dp[i][type] = max(f(i - 1, 0), f(i - 1, 2)) + d[i][type];
  }
  else if(type == 2){
    dp[i][type] = max(f(i - 1, 0), f(i - 1, 1)) + d[i][type];
  }

  return dp[i][type];
}
كود معدل: جعله ديناميكياً ليدعم K من الأنشطة
  // الاستغناء عن جمل الـ if بـ حلقة تكرار (For loop)
  int best_prev = 0;
  
  for (int prev_type = 0; prev_type < 3; prev_type++) {
      // شرط: يمنع اختيار نفس نشاط الأمس
      if (prev_type != type) {
          best_prev = max(best_prev, f(i - 1, prev_type));
      }
  }
  
  dp[i][type] = best_prev + d[i][type];
  return dp[i][type];
}
هنا يتم تطبيق الشرط الرئيسي للمسألة (لا يجوز ممارسة نفس النشاط ليومين متتاليين).
لذلك، إذا أردت اختيار النشاط 0 اليوم، فيجب أن أبحث عن الأفضل (max) من مسارات الأمس التي تنتهي بالنشاط 1 أو النشاط 2 حصراً، ثم أجمع معها نقاط اليوم.
تغير المعطيات
كتابة شروط if / else يدوياً تعمل جيداً لأن عدد الأنشطة هنا 3 فقط. ولكن في أمثلة مشابهة حيث يسمح لك بالاختيار من بين K من الأنشطة (مثلاً 10 أنشطة)، لا يمكننا كتابة 10 شروط يدوياً! بل نستخدم حلقة تكرار كما يظهر في الكود المُعدل للمرور على كل الأنشطة السابقة وتخطي المطابق للنشاط الحالي.

التهيئة والدالة الرئيسية (Main)

int main() {
  
    // تهيئة مصفوفة 2D بحلقتين متداخلتين
    for (int i = 0; i < 200000; i++) {
    for (int j = 0; j < 3; j++) {
          dp[i][j] = -1;
    }
    }

    int n; cin >> n;

    for (int i = 0; i < n; i++) {
    for (int j = 0; j < 3; j++) {
      cin >> d[i][j];
    }
    }

    // إيجاد الحد الأقصى بين جميع النهايات الممكنة لليوم الأخير
    cout << max(max(f(n - 1, 0), f(n - 1, 1)), f(n - 1, 2)) << "\n";
}
كود معدل: تحويل الكود إلى أسلوب (Bottom-Up Iterative DP)
int main() {
    /* ... (قراءة المعطيات) ... */

    // Base Case
    dp[0][0] = d[0][0]; dp[0][1] = d[0][1]; dp[0][2] = d[0][2];

    // Transition Loop
    for (int i = 1; i < n; i++) {
        dp[i][0] = max(dp[i-1][1], dp[i-1][2]) + d[i][0];
        dp[i][1] = max(dp[i-1][0], dp[i-1][2]) + d[i][1];
        dp[i][2] = max(dp[i-1][0], dp[i-1][1]) + d[i][2];
    }

    cout << max(max(dp[n-1][0], dp[n-1][1]), dp[n-1][2]) << "\n";
}
نظرًا لأن المصفوفة الآن ثنائية الأبعاد، استخدمنا حلقتين متداخلتين لتهيئتها بـ -1، وكذلك لقراءة مصفوفة المدخلات. وفي السطر الأخير، بما أننا لا نعرف أي نشاط سينتهي به المطاف في اليوم الأخير بأكبر مجموع، قمنا باستدعاء الدالة ثلاث مرات (لكل نشاط) واختيار القيمة الأكبر بينهم.
تغير المعطيات
في المنافسات الاحترافية، لتفادي أخطاء امتلاء مكدس الاستدعاءات (Stack Overflow)، يتم أحياناً تحويل المسألة من طريقة Top-Down (بالدوال التراجعية) إلى أسلوب Bottom-Up. كما يظهر في الكود المجاور، يتم الاستغناء عن دالة f(i) تماماً، ونقوم بملء قيم الـ dp يدوياً تصاعدياً من اليوم الأول وحتى الأخير داخل حلقة التكرار في دالة الـ main.
