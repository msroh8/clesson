التحليل الهيكلي لخوارزميات البرمجة التنافسية

مسألة: Painting the Fence (Intervals)
هذا المستند يحلل مسألة تعتمد على منطق الفترات والتقاطعات (Intervals & Intersections). عند الضغط على أمثلة مشابهة، سيتغير الكود ليوضح كيف يمكن تكييف هذا الأسلوب لمسائل أخرى مثل التعامل مع عدد غير محدود من الفترات أو الترتيب.
1

التعامل مع الملفات (USACO I/O)

#include <iostream>

using namespace std;

int main() {
  // USACO هذين السطرين فقد لأخذ المدخلات وطباعة المخرجات في موقع 
  freopen("paint.in", "r", stdin);
  freopen("paint.out", "w", stdout);
كود معدل: منصات مثل Codeforces أو LeetCode
#include <iostream>

using namespace std;

int main() {
  // في المنصات الأخرى، الإدخال والإخراج يكون من الـ Console مباشرة
  // لا حاجة لـ freopen، بل نستخدم هذه الأسطر للتسريع فقط:
  ios::sync_with_stdio(false);
  cin.tie(nullptr);
تاريخياً، كانت مسابقة USACO تتطلب من المتسابقين قراءة المدخلات من ملف نصي (مثل paint.in) وكتابة المخرجات في ملف آخر (مثل paint.out). الدالة freopen تقوم بإعادة توجيه الدخل القياسي stdin والخرج القياسي stdout للتعامل مع هذه الملفات تلقائياً عند استخدام cin و cout.
تغير المعطيات
معظم المنصات الحديثة للبرمجة التنافسية (مثل Codeforces أو النسخ الأحدث من USACO) لم تعد تستخدم هذا الأسلوب وتعتمد على الإدخال القياسي المباشر. في هذه الحالة، نزيل freopen تماماً، ونكتفي بوضع أسطر تسريع الإدخال والإخراج القياسية إذا لزم الأمر.
2

قراءة إحداثيات الفترات (Interval Coordinates)

  int a_l , a_r, b_l , b_r;
  
  cin >> a_l >> a_r >> b_l >> b_r;
كود معدل: التعامل مع N من الفترات (Array of Structs/Pairs)
  // إذا كان لدينا عدد غير محدد (N) من الفترات وليس اثنتين فقط
  int n; cin >> n;
  vector<pair<int, int>> intervals(n);
  
  for(int i = 0; i < n; i++) {
      cin >> intervals[i].first >> intervals[i].second;
  }
المتغيرات هنا تعبر عن فترة معينة على خط الأعداد. a_l هو بداية الفترة A (اليسار)، و a_r هو نهايتها (اليمين). وبالمثل للفترة B. تقرأ هذه المتغيرات تباعاً من الدخل.
تغير المعطيات
في المسائل الأكثر تقدماً (مثل Merge Intervals)، لا يقتصر الأمر على فترتين فقط بل N من الفترات. في هذه الحالة، لن نعرّف متغيرات منفصلة، بل سنستخدم مصفوفة أو vector من نوع pair<int, int> لتخزين بداية ونهاية كل فترة.
3

إيجاد منطقة التقاطع (Intersection)

  // b و a يمثل التقاطع بين c
  int c_l = max(a_l, b_l), c_r = min(a_r, b_r);
كود معدل: تقاطع مستطيلات (2D Intersection)
  // لو كنا نبحث عن تقاطع مستطيلين (2D) بدلاً من خطين (1D)
  // نطبق نفس المنطق لمحور X ولمحور Y بشكل منفصل
  int inter_x_l = max(rect1_x1, rect2_x1);
  int inter_x_r = min(rect1_x2, rect2_x2);
  
  int inter_y_b = max(rect1_y1, rect2_y1);
  int inter_y_t = min(rect1_y2, rect2_y2);
هذا السطر هو قلب المنطق الرياضي! لكي نجد الفترة c التي تُمثل التقاطع بين a و b:
- بداية التقاطع c_l هي الأكبر (max) بين بدايتي الفترتين.
- نهاية التقاطع c_r هي الأصغر (min) بين نهايتي الفترتين.
تغير المعطيات
في مسائل الهندسة الحسابية (Computational Geometry)، قد يُطلب منك إيجاد مساحة التقاطع بين مستطيلين وليس مجرد فترتين. المفاجأة أن نفس المنطق تماماً يُستخدم! كل ما عليك فعله هو تطبيقه مرتين: مرة لإيجاد تقاطع الإحداثيات السينية (محور X)، ومرة للإحداثيات الصادية (محور Y).
4

حساب الأطوال (Lengths Calculation)

  int total = (a_r - a_l) + (b_r - b_l);
  int intersection = (c_r - c_l);
كود معدل: حماية ضد التقاطع الوهمي (Negative Length)
  int total = (a_r - a_l) + (b_r - b_l);
  
  // إذا كانت c_r أقل من c_l، فالتقاطع غير موجود فعلياً (طول سالب)
  // نستخدم max(0, length) لمنع القيم السالبة
  int intersection = max(0, c_r - c_l);
نحسب هنا المجموع الإجمالي لأطوال الفترتين كأن كل منهما منفصلة تماماً (الطول = النهاية - البداية).
ثم نحسب طول فترة "التقاطع" المحتملة التي أوجدناها في الخطوة السابقة.
تغير المعطيات
ماذا لو كانت الفترتان متباعدتين جداً؟ في هذه الحالة ستكون c_l > c_r، وسيكون طول التقاطع سالباً!.
بدلاً من معالجة ذلك بشروط لاحقة (كما فعلت في الجزء 5)، يقوم المحترفون عادةً بحساب طول التقاطع هكذا: max(0, c_r - c_l). إذا كان الطول سالباً (وهمياً)، يصبح صفراً مباشرة.
5

التحقق من التباعد وطباعة الحل (Final Logic)

  // يتقاطعان b و a الآن يجب أن نتحقق ما إذا 
  if(b_l > a_r || a_l > b_r){
    cout << total << "\n";
  }
  else{
    cout << total - intersection << "\n";
  }
}
كود معدل: تبسيط المنطق باستخدام حماية التقاطع (Step 4)
  // إذا استخدمنا max(0, c_r - c_l) في الخطوة السابقة
  // فلا حاجة لأي شروط if/else هنا إطلاقاً!
  
  // نطبع المجموع الكلي ناقص التقاطع (سواء كان 0 أو قيمة فعلية)
  cout << total - intersection << "\n";
}
هنا نتحقق يدوياً هل الفترتان متباعدتان؟
الشرط b_l > a_r يعني أن B تقع بالكامل يمين A. والشرط المعاكس يعني أنها يسارها. إذا تحقق أحدهما فهذا يعني لا يوجد تقاطع، فنطبع مجموع الأطوال الكامل total.
أما إذا كان هناك تقاطع (حالة الـ else)، فنحن جمعنا منطقة التقاطع مرتين! لذا يجب طرحها مرة واحدة: total - intersection.
تغير المعطيات
كما ذكرنا في الخطوة السابقة، إذا أمّنت حساب التقاطع وجعلته لا ينزل عن الصفر max(0, ...)، يمكنك الاستغناء عن كتلة الـ if-else بأكملها هنا. ستطبع دائماً total - intersection. إذا كانت الفترتان متباعدتين، سيكون التقاطع 0، وبالتالي ستطبع الـ total بشكل صحيح دون الحاجة لشرط خاص. هذا الأسلوب يقلل من احتمالية الأخطاء (Bugs).
