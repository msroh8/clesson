الدرس الأول: فهم المتجر وقواعد الرضا

تحليل مسألة "الرغبات والاحتياجات" وتبسيط المتغيرات المرفقة.

قاموس المصطلحات (لنبسطها معاً!) 📖

أنت في متجر كمبيوتر وتريد ترقية جهازك. هدفك هو جمع عدد معين من نقاط الأداء بأقل عدد من المشتريات. دعونا نترجم رموز المسألة:

K الهدف من النقاط:

مجموع نقاط الأداء الذي يجب أن يصله جهازك كحد أدنى (مثلاً K=5 نقاط).

N أقسام المتجر:

كم قسماً مختلفاً متاحاً لك لتشتري منه (منتج 1، منتج 2...).

A القطع العادية (الاحتياجات):

القطع الإجبارية التي يجب شراؤها أولاً. كل قطعة عادية تعطيك نقطة أداء واحدة.

B القطع الخارقة (الرغبات):

بعد تجاوز عدد (A)، كل قطعة إضافية تشتريها ستعطيك نقطتين مضاعفتين!

الدرس الثاني: الفخ الجشع (Greedy)

لماذا تفشل الحلول المباشرة، وكيف نغير تفكيرنا؟

⚠️ لماذا يفشل الحل الجشع (Greedy)؟

التفكير المبتدئ هو: "سأبحث عن العرض الذي يعطيني أكبر عدد من القطع الذهبية وأشتريه فوراً لكي أنتهي بسرعة".

تخيل هذا السيناريو المخادع:
تحتاج لجمع K = 5 نقاط فقط.
العرض الأول: A = 1000, B = 1005 (يعطيك 5 قطع ذهبية، لكن الرسوم تجبرك على شراء 1000 قطعة عادية أولاً!).
العرض الثاني: A = 0, B = 5 (رسوم الدخول صفر، والقطع الخمس كلها ذهبية).

الخوارزمية الجشعة قد تختار العرض الأول لمجرد أن فرقه كبير، وتورطك بشراء 1003 قطع! بينما الحل الأمثل هو شراء 3 قطع من العرض الثاني.

نظرية الاستثمار: تحويل المتغيرات 📈

لتجاوز الفخ، سنقوم بإعادة تسمية المتغيرات ليتعامل الكمبيوتر مع كل قسم كـ "مشروع استثماري":

  • التكلفة الإجبارية (w): تساوي رياضياً A.
    هذا هو الثمن الذي "نجبر" على دفعه. نعرّف متغير w = A.
  • المكافأة الصافية (v): تساوي رياضياً B - A.
    عدد القطع الذهبية التي ستفتح لنا. نعرّف متغير v = max(0, B - A). (ملاحظة: نستخدم max لحماية الخوارزمية من المنتجات الخاسرة).

الدرس الثالث: خوارزمية حقيبة الظهر (0-1 Knapsack)

استخدام البرمجة الديناميكية (DP) لتذكر أفضل الاستثمارات.

حجز الذاكرة (DP Array) 🧠

سنعرّف مصفوفة في الذاكرة باسم dp لحفظ أفضل العروض.

المعادلة الأساسية في الذاكرة:
dp[j] = v
تعني: "إذا دفعت بالضبط j من القطع العادية كرسوم، فإن أقصى عدد من المكافآت الذهبية التي أمتلكها هو v."

كيف نملأ المصفوفة برمجياً؟ (Loop Logic)

  • نعرّف مصفوفة بحجم K + 1 ونملأها بالرقم -1 (ومعناه: لم أصل لهذا المجموع قط).
  • نعطي الخانة الأولى قيمة صفر dp[0] = 0 (دفع 0 يعطي 0 مكافأة).
  • نقوم بعمل حلقة (Loop) تمر على كل المنتجات واحداً تلو الآخر.
  • بداخلها، نقوم بعمل حلقة ثانية تمر على الذاكرة j بشكل عكسي (من K نزولاً للتكلفة w).
  • نسأل: هل الخانة السابقة j - w تم الوصول إليها؟ (أي ليست -1).
  • إذا نعم، نحدّث الخانة الحالية dp[j] لتأخذ القيمة الأكبر بين: قيمتها السابقة، أو قيمة الخانة السابقة + مكافأة المنتج الجديد.

الدرس الرابع: معادلة حصد النتائج

كيف يحسب الكمبيوتر الإجابة النهائية من المصفوفة؟

بعد امتلاء مصفوفة الذاكرة dp، أصبح لدينا في كل خانة w عدد من القطع الذهبية v.
بما أننا دفعنا w قطعة (حصلنا على w نقطة)، فإن النقاط المتبقية للوصول للهدف K هي: rem = max(0, K - w).

العمليات الرياضية لحساب عدد القطع:

  • الحالة الأولى: المكافآت (v) تكفينا وأكثر!

    كل قطعة ذهبية تعطي نقطتين. إذن إذا كان v * 2 ≥ rem، فنحن لا نحتاج لشراء كل القطع الذهبية، بل سنشتري فقط ما نحتاجه لنغطي النقاط المتبقية.

    القطع الإجمالية = w + (rem + 1) / 2.

  • الحالة الثانية: المكافآت (v) لا تكفي!

    إذا كان v * 2 < rem، نضطر لشراء كل القطع الذهبية (v)، ثم نكمل ما تبقى من النقاط بشراء قطع عادية باهظة (بنقطة واحدة).

    القطع الإجمالية بتبسيط رياضي = K - v.

الدرس الخامس: تفكيك الكود الأساسي

كيف نترجم كل هذا الشرح إلى كود C++ نظيف ومنظم؟ (النسخة المعتمدة والمصححة).

اضغط على علامة التعجب ! بجوار الكود لتفهم وظيفة كل كتلة برمجية تم إضافتها لحماية الكود.

  • vector<int> dp(K + 1, -1);
    مصفوفة الذاكرة. حجمها K+1 لأننا لا نحتاج لاستثمار يتجاوز K. بدأت بـ -1 لتدل على عدم الزيارة.
  • cin >> A[i] >> B[i];
    الترتيب الصحيح لقراءة التكلفة أولاً ثم الإجمالي.
  • int v = max(0, B[i] - A[i]);
    حماية: إذا كان المنتج خاسراً، نتجاهل الخسارة ونجعل المكسب 0 بدلاً من قيمة سالبة تخرب الذاكرة.
  • dp[j] = min(K, max(...))
    🛡️ حركة المحترفين لمنع الـ Overflow: نضع حداً أقصى min(K, ...) لمنع الأرقام من الانفجار بسبب المكافآت الضخمة.
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    // تسريع الإدخال والإخراج لتجنب Time Limit
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N, K;
    if (!(cin >> N >> K)) return 0;
    
    vector<int> A(N), B(N);
    // قراءة A (التكلفة) ثم B (الإجمالي)
    for (int i = 0; i < N; i++) cin >> A[i] >> B[i];

    // مصفوفة الذاكرة (0-1 Knapsack)
    vector<int> dp(K + 1, -1);
    dp[0] = 0;

    for (int i = 0; i < N; i++) {
        int w = A[i];                // التكلفة (الوزن)
        int v = max(0, B[i] - A[i]); // المكسب الصافي (محمي من السالب)
        if (w > K) continue;         // تجاهل الاستثمار الخاسر

        // تحديث الذاكرة من الخلف للأمام
        for (int j = K; j >= w; j--) {
            if (dp[j - w] != -1) {
                // حماية الذواكر من الـ Overflow بوضع الحد K
                dp[j] = min(K, max(dp[j], dp[j - w] + v));
            }
        }
    }

    int ans = K; // أسوأ سيناريو: شراء K قطعة عادية بدون أي عروض
    
    // مرحلة حصد النتائج: نمر على كل عروض الذاكرة
    for (int w = 0; w <= K; w++) {
        if (dp[w] != -1) {
            int v = dp[w];
            int rem = max(0, K - w); // النقاط المتبقية للهدف
            int pieces;
            
            if (v * 2 >= rem) pieces = w + (rem + 1) / 2; // المكافأة تكفي وتزيد
            else              pieces = K - v;             // المكافأة نفدت
            
            ans = min(ans, pieces); // حفظ أقل عدد ممكن من القطع
        }
    }
    cout << ans << "\n";
    return 0;
}

الدرس السادس: المشغل التفاعلي للكود

أدخل قيمك الخاصة، وشاهد بعينيك كيف يترجم سطر الكود إلى حركة فعلية في الذاكرة بالتزامن مع المحاكاة!

⚙️ لوحة إعداد البيانات

الهدف (K): 5

المتغيرات في الذاكرة الآن

المنتج (i) -
التكلفة (w) -
المكافأة (v) -
المؤشر (j) -
أقل قطع (ans) -
#include <iostream>
#include <vector>
using namespace std;
int main() {
  int N = 3, K = 5;
  vector<int> A, B; // Filled dynamically
  vector<int> dp(K + 1, -1);
  dp[0] = 0;
  // 0-1 Knapsack DP
  for (int i = 0; i < N; i++) {
  int w = A[i], v = max(0, B[i] - A[i]);
  if (w > K) continue;
  for (int j = K; j >= w; j--) {
  if (dp[j - w] != -1) {
  dp[j] = min(K, max(dp[j], dp[j - w] + v));
  }
  }
  }
  // Calculate minimum pieces
  int ans = K;
  for (int w = 0; w <= K; w++) {
  if (dp[w] != -1) {
  int rem = max(0, K - w), pieces;
  if (dp[w] * 2 >= rem) pieces = w + (rem+1)/2;
  else pieces = K - dp[w];
  ans = min(ans, pieces);
  }
  }
  cout << ans;
}

🧠 التتبع الذهني للسطر المُظلل

🗄️ الذاكرة الفعلية (مصفوفة dp)

كل خانة تمثل التكلفة المدفوعة (j). الرقم بداخلها هو المكافأة المفتوحة (v). القيمة -1 تعني مساراً مستحيلاً.

الدرس السابع: عبقرية التعامل مع القيود ⚠️

كيف تمكن كودنا من اختراق جميع القيود الصعبة وحصد 100 نقطة؟

القيود المرفقة في المسألة تضع المبرمج أمام تحديات ضخمة. دعونا نرى كيف نجونا منها:

1. قيد الوقت (N, K ≤ 5000)

التعقيد الزمني لخوارزميتنا هو O(N × K). لو ضربنا 5000 في 5000، الناتج هو 25 مليون عملية فقط. الكمبيوتر ينفذ هذا العدد في حوالي 0.05 ثانية. نمر من قيد الـ 1 ثانية بسهولة تامة!

2. فخ الذاكرة الخبيث A ≤ 10^6

لو أنشأنا مصفوفة dp بحجم A (مليون)، لانفجرت الذاكرة وحصلنا على (Memory Limit). العبقرية: نحن نعلم أننا لا نحتاج لاستثمار يتجاوز K (لأن K=5000 كحد أقصى). السطر if (w > K) continue; يحمينا من حجز مصفوفات عملاقة، ويبقي حجم مصفوفتنا صغيراً جداً (5000 خانة فقط)!

3. فخ انفجار الأرقام (Integer Overflow)

المكافآت (v) قد تصل لمليون أيضاً وتتراكم وتتخطى سعة متغير الـ int! العبقرية: السطر min(K, max(...)). نحن لا يهمنا تخزين أرقام أكبر من K. إذا تجاوزت المكافأة قيمة K المطلوبة، فإن الدالة min تقصها وتثبتها عند K. حماية 100% للذاكرة.

الدرس الثامن: أخطاء تجعل كودك لا يعمل 🛑

لماذا حصل الكود السابق على 0 نقطة في التقييم؟ وكيف تنجو من فخوخ المنصات التنافسية؟

الخطأ الأول

1. القاتل الصامت: ترتيب الإدخال المعكوس

في الشروحات النظرية أحياناً يتم ذكر الإجمالي B ثم التكلفة A. ولكن في المنصات التنافسية، دائماً ما تعطى التكلفة أولاً ثم الإجمالي. الكود القديم قرأها بالعكس مما جعل حسابات التكلفة والعائد خاطئة تماماً!

cin >> B[i] >> A[i]; // خطأ: يعكس القيم
cin >> A[i] >> B[i]; // الصح: قراءة التكلفة ثم الإجمالي
الخطأ الثاني

2. فخ المنتجات الخاسرة (القيم السالبة)

بعض الاختبارات الخبيثة تضع منتجات تكلفتها أعلى من عائدها الإجمالي، مما يجعل الطرح B - A يعطي قيمة سالبة. القيم السالبة تدمر مصفوفة الـ DP وتجعلها تبحث عن مسارات وهمية. يجب وضع حد أدنى للصفر.

int v = B[i] - A[i];
int v = max(0, B[i] - A[i]); // الصح: تصفير الخسارة
الخطأ الثالث

3. تجاوز سقف الذاكرة (Memory Overflow)

ماذا لو كان أحد العروض يعطي مليون نقطة مكافأة، وأنت تحتاج لـ 5 نقاط فقط؟ تراكم الأرقام الضخمة في الذاكرة قد يسبب Overflow. نحن لا نحتاج لمكافآت تتجاوز هدفنا الأساسي K، لذا يجب قصها ووضع سقف لها.

dp[j] = max(dp[j], dp[j - w] + v);
dp[j] = min(K, max(dp[j], dp[j - w] + v)); // الصح: السقف هو K
الخطأ الرابع

4. بطء أداء الإدخال والإخراج (Time Limit)

عندما تحتوي المسألة على مئات أو آلاف الاختبارات، فإن أوامر cin, cout الافتراضية في C++ تكون بطيئة جداً وتتسبب في رسالة Time Limit Exceeded. يجب دائماً إيقاف المزامنة لتسريع الأداء.

ios_base::sync_with_stdio(false);
cin.tie(NULL);
