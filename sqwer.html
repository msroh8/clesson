المربع الكامل (Perfect Square)

إعداد وتصميم

المدرب / محمد مجممي

الدرس الأول: تحليل المسألة وفخ القيود

قراءة ما بين السطور في مسائل المسابقات.

المطلوب في المسألة 🎯

يُعطى لك عدد صحيح N. ويجب عليك تحديد ما إذا كان يوجد عدد صحيح x بحيث تتحقق المعادلة التالية:

x^2 = N

باختصار، المسألة تسألك: هل الرقم N هو "مربع كامل" (Perfect Square) أم لا؟
إذا كان كذلك، اطبع Yes، وإلا اطبع No.

الأمثلة التوضيحية (Examples):

  • N = 9 ← Yes (لأن 3^2 = 9)
  • N = 8 ← No (لا يوجد رقم صحيح تضربه في نفسه ليعطيك 8)

⚠️ قراءة القيود الفلكية (Constraints Analysis)

في المسابقات، القيود هي التي تخفي الفخاخ. لاحظ القيد الموجود في ملف المسألة:

-10^5 ≤ N ≤ 10^5

الفخ المخفي: الأرقام السالبة!

الحد الأدنى للقيد هو -10^5. هذا يعني أن واضع المسألة سيدخل لك أرقاماً سالبة (مثل -9). رياضياً، لا يوجد أي عدد صحيح تضربه في نفسه ليعطيك رقماً سالباً! إذا حاولت تمرير رقم سالب إلى دالة الجذر التربيعي sqrt() مباشرة دون التحقق، سيقوم البرنامج بإصدار خطأ (Domain Error) أو إرجاع قيمة وهمية (NaN) وسيفشل الكود في المسابقة.

الدرس الثاني: الاستراتيجية الرياضية

كيف نفحص المربع الكامل برمجياً بأكثر الطرق كفاءة وأماناً؟

خطوات الحل المثالية 🛠️

1️⃣ حماية الكود (الجدار الواقي)

أول سطر يجب كتابته هو حماية من الأرقام السالبة. إذا كان N < 0، نطبع No مباشرة وننهي البرنامج.

2️⃣ إيجاد الجذر التقريبي

نستخدم دالة الجذر التربيعي المدمجة في C++ وهي sqrt(N). لكن هذه الدالة تعيد أرقاماً عشرية (Double). مثلاً جذر 8 هو 2.828. لذا نقوم بتخزين الناتج داخل متغير صحيح int، مما يجعل الكمبيوتر يحذف الكسر العشري ويحتفظ بالرقم الصحيح فقط (2).

3️⃣ اختبار الإثبات (The Proof)

الآن لدينا رقم صحيح أسميناه root. لنتأكد أنه الجواب الصحيح، نقوم بضربه في نفسه ونتحقق مما إذا كان يساوي N الأصلي:
if (root * root == N)
إذا ساوى N إذن هو مربع كامل!

المخطط الانسيابي التفاعلي (Interactive Flowchart)

الدرس الرابع: تفكيك الكود الأساسي

ترجمة المخطط الانسيابي إلى كود C++ نظيف واحترافي.

اضغط على علامة التعجب ! بجوار الكود لتفهم وظيفة كل كتلة برمجية بدقة.

  • #include <cmath>
    استدعاء المكتبة الرياضية التي تحتوي على دالة الجذور sqrt().
  • if (N < 0)
    خطوة الأمان لتصفية الأرقام السالبة وطباعة No مباشرة.
  • int root = sqrt(N);
    حساب الجذر. تخزينه في int يقوم بحذف الكسور. مثلاً جذر 8 (2.82) يصبح (2).
  • if (root * root == N)
    الإثبات: هل 2 × 2 يساوي 8؟ لا. إذن يطبع No. هل 3 × 3 يساوي 9؟ نعم. إذن يطبع Yes.
#include <iostream>
#include <cmath>  // مكتبة الرياضيات ضرورية هنا
using namespace std;

int main() {
    int N;
    cin >> N;

    // 1. الجدار الواقي من الفخ (الأرقام السالبة)
    if (N < 0) {
        cout << "No\n";
        return 0; // ننهي البرنامج هنا
    }

    // 2. إيجاد الجذر التقريبي وحذف الكسور
    int root = sqrt(N);

    // 3. الإثبات الرياضي
    if (root * root == N) {
        cout << "Yes\n";
    } else {
        cout << "No\n";
    }

    return 0;
}

الدرس السادس: قيود متقدمة وحلول بديلة

ماذا لو كان الرقم N فلكياً في مسائل أخرى؟ إليك كيف تتصرف كالمحترفين!

🔥 القيد المتقدم: N \le 10^{18}

في مسألتنا الحالية، القيد كان صغيراً 10^5 لذلك استخدمنا int بأمان. لكن لو تغير القيد إلى 10^{18} في مسابقات أعلى (Div.2):

  • نوع البيانات: يجب تغيير جميع المتغيرات إلى long long.
  • مشكلة دقة الجذور (Precision): دالة sqrt() قد تخطئ وتفقد الدقة مع الأرقام التي تقترب من 10^{18} بسبب مشاكل تمثيل الفواصل العشرية في الكمبيوتر (Floating Point Precision). قد تحسب جذر رقم كبير على أنه 999999999 بدلاً من 1000000000!

الحل الآمن (استخدام دالة التقريب):

نستخدم دالة round() لضمان تصحيح الخلل العشري، واستخدام sqrtl() المخصصة للأرقام الكبيرة.

long long N; cin >> N;
long long root = round(sqrtl(N)); // الجدار الواقي للقيود الضخمة
if (root * root == N) cout << "Yes";

🔍 البديل العبقري: البحث الثنائي (Binary Search)

ماذا لو منعك واضع المسألة من استخدام مكتبة <cmath> تماماً؟ يمكنك بناء دالة الجذر التربيعي الخاصة بك باستخدام البحث الثنائي!

نحدد النطاق من L = 0 إلى R = 10^9 (لأن أقصى جذر لـ 10^{18} هو 10^9). ثم نفحص نقطة المنتصف mid. 

long long L = 0, R = 1000000000, ans = -1;
while(L <= R) {
    long long mid = L + (R - L) / 2;
    if (mid * mid == N) { ans = mid; break; }
    if (mid * mid < N) L = mid + 1;
    else R = mid - 1;
}
if(ans != -1) cout << "Yes"; else cout << "No";

هذا الكود سيعمل في O(log N) ويعتبر من أأمن وأقوى الطرق في عالم الـ Competitive Programming.
