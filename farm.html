الدرس الأول: فهم المزرعة والهدف المطلوب

تحليل مسألة "الزراعة (Farming)" وفك شفرة المتغيرات.

قاموس المصطلحات 📖

لدينا مزرعة كبيرة مقسمة إلى شبكة مربعة. يريد صديقان اختيار "مربعين" من هذه المزرعة لحصادهما. الشرط الأساسي ألا يتقاطع المربعان. لنتعرف على المتغيرات:

N حجم المزرعة الكلية:

المزرعة عبارة عن شبكة أبعادها N × N. الخلية الواحدة تحتوي على قيمة المحصول.

K حجم المربع المطلوب:

كل صديق يريد حصاد مربع فرعي أبعاده K × K בדיוק.

Gi,j قيمة المحصول (موجب أو سالب!):

كل خلية لها قيمة، إذا كانت موجبة فهي أرباح، وإذا كانت سالبة فهي محاصيل فاسدة تسبب خسارة. هدفنا هو إيجاد مربعين غير متقاطعين يكون مجموع خلاياهما أكبر ما يمكن.

الدرس الثاني: الفخ البطيء (Brute Force)

كيف يفكر المبتدئون؟ ولماذا يتسبب هذا التفكير في تجاوز الوقت (Time Limit)؟

🐢 الحل التقليدي البطيء جداً

التفكير البديهي هو تجربة كل الاحتمالات الممكنة:

  • نختار المربع الأول (يحتاج لتحديد نقطة البداية x1, y1).
  • نختار المربع الثاني (يحتاج لتحديد نقطة البداية x2, y2).
  • نتأكد أنهما لا يتقاطعان.
  • نقوم بجمع القيم داخل المربعين بمرور (Loop) على جميع خلاياهما.
لماذا هذا كارثة برمجية؟

حجم المزرعة N يصل إلى 500. لاختيار المربعين وحساب المجموع، سيقوم الكمبيوتر بعمليات تقارب O(N6).
5006 = 15,625,000,000,000,000 عملية!
الكمبيوتر يحتاج إلى سنين لحل هذه المسألة، بينما الوقت المسموح هو 1 ثانية فقط!

الدرس الثالث: سرعة البرق (2D Prefix Sum)

التقنية الأولى: كيف نحسب مجموع أي مربع في لحظة واحدة O(1)؟

⚡ مصفوفة المجاميع التراكمية ثنائية الأبعاد

للتخلص من بطء حساب المجموع في كل مرة، نقوم بتجهيز مصفوفة مسبقة اسمها pref. كل خلية فيها تحفظ مجموع المستطيل الممتد من أعلى اليسار (0,0) إلى هذه الخلية.

معادلة البناء السحرية:

pref[i][j] = G[i][j] + pref[i-1][j] + pref[i][j-1] - pref[i-1][j-1]

نطرح الجزء المتقاطع pref[i-1][j-1] لأننا جمعناه مرتين.

معادلة الاستخراج في O(1):

الآن، إذا أردنا مجموع مربع حجمه K × K ينتهي عند الزاوية السفلية اليمنى (i, j)، نستخدم هذه المعادلة الفورية:

sum = pref[i][j] - pref[i-K][j] - pref[i][j-K] + pref[i-K][j-K]

بهذه المعادلة، أصبحنا نعرف مجموع أي مربع فوراً بدون أي Loops!

الدرس الرابع: خط الفصل السحري

التقنية الثانية: كيف نضمن عدم تقاطع المربعين بذكاء؟

هناك قاعدة هندسية ثابتة: أي مستطيلين/مربعين لا يتقاطعان في شبكة، يمكن دائماً الفصل بينهما بخط مستقيم (إما خط أُفقي أو خط عمودي) يمر من أول الشبكة لآخرها.

الفصل الأُفقي (Horizontal Split)

نمرر خطاً أفقياً ونبحث عن: أفضل مربع في المنطقة العلوية + أفضل مربع في المنطقة السفلية.

الفصل العمودي (Vertical Split)

نمرر خطاً عمودياً ونبحث عن: أفضل مربع في المنطقة اليمنى + أفضل مربع في المنطقة اليسرى.

الخطة البرمجية للتقسيم:

  • 1. نجهز 4 مصفوفات تحفظ أفضل مجموع للمربعات في الاتجاهات الأربعة: top_max, bottom_max, left_max, right_max.
  • 2. نجرّب كل خطوط القص الأفقية الممكنة، ونحسب:
    ans = max(ans, top_max[i] + bottom_max[i+1])
  • 3. نجرّب كل خطوط القص العمودية الممكنة، ونحسب:
    ans = max(ans, left_max[j] + right_max[j+1])

الدرس الخامس: الكود الأساسي للمحترفين

النسخة المعتمدة والمحمية ضد التجاوزات باستخدام C++.

اضغط على علامة التعجب ! بجوار الكود لتفهم وظيفة كل كتلة برمجية. (استخدمنا 1-based indexing لتسهيل الحسابات).

  • long long: استخدمناها لأن القيم تصل إلى $10^9$، والمجموع قد يصل لـ $2.5 \times 10^{14}$ مما يكسر الـ int.
  • -1e18 (-INF): القيمة الابتدائية. لا تستخدم صفر أبداً! لأن أفضل مزرعة قد تكون كلها قيم سالبة.
  • auto get_sum: دالة سريعة (Lambda) تستخدم مصفوفة pref لجلب مجموع أي مربع KxK في لحظة واحدة O(1).
  • الفصل (Split): في نهاية الكود، اللوب الأول يفصل الشبكة أفقياً (top + bottom)، واللوب الثاني يفصل عمودياً (left + right).
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

// حماية من القيم السالبة الضخمة
const long long INF = 1e18; 

int main() {
    // تسريع القراءة لتخطي Time Limit
    ios_base::sync_with_stdio(false); cin.tie(NULL);
    
    int N, K;
    if (!(cin >> N >> K)) return 0;

    // مصفوفة البيانات ومصفوفة التراكم (1-based indexing)
    vector<vector<long long>> grid(N + 1, vector<long long>(N + 1, 0));
    vector<vector<long long>> pref(N + 1, vector<long long>(N + 1, 0));

    // قراءة البيانات وبناء مصفوفة المجاميع التراكمية
    for (int i = 1; i <= N; i++) {
        for (int j = 1; j <= N; j++) {
            cin >> grid[i][j];
            pref[i][j] = grid[i][j] + pref[i - 1][j] + pref[i][j - 1] - pref[i - 1][j - 1];
        }
    }

    // دالة لاستخراج مجموع مربع KxK ينتهي عند (i, j) في O(1)
    auto get_sum = [&](int i, int j) {
        return pref[i][j] - pref[i - K][j] - pref[i][j - K] + pref[i - K][j - K];
    };

    // مصفوفات لحفظ أفضل المربعات من الاتجاهات الأربعة
    vector<long long> top_max(N + 2, -INF), bottom_max(N + 2, -INF);
    vector<long long> left_max(N + 2, -INF), right_max(N + 2, -INF);

    // 1. حساب أفضل مربع من الأعلى
    long long cur_max_top = -INF;
    for (int i = K; i <= N; i++) {
        for (int j = K; j <= N; j++) {
            cur_max_top = max(cur_max_top, get_sum(i, j));
        }
        top_max[i] = cur_max_top;
    }

    // 2. حساب أفضل مربع من اليسار
    long long cur_max_left = -INF;
    for (int j = K; j <= N; j++) {
        for (int i = K; i <= N; i++) {
            cur_max_left = max(cur_max_left, get_sum(i, j));
        }
        left_max[j] = cur_max_left;
    }

    // 3. حساب أفضل مربع من الأسفل
    long long cur_max_bottom = -INF;
    for (int i = N - K + 1; i >= 1; i--) {
        for (int j = K; j <= N; j++) {
            cur_max_bottom = max(cur_max_bottom, get_sum(i + K - 1, j));
        }
        bottom_max[i] = cur_max_bottom;
    }

    // 4. حساب أفضل مربع من اليمين
    long long cur_max_right = -INF;
    for (int j = N - K + 1; j >= 1; j--) {
        for (int i = K; i <= N; i++) {
            cur_max_right = max(cur_max_right, get_sum(i, j + K - 1));
        }
        right_max[j] = cur_max_right;
    }

    long long ans = -INF; // الجواب النهائي
    
    // التقسيم الأفقي (نجرب كل خطوط القص الأفقية)
    for (int i = K; i <= N - K; i++) {
        ans = max(ans, top_max[i] + bottom_max[i + 1]);
    }
    
    // التقسيم العمودي (نجرب كل خطوط القص العمودية)
    for (int j = K; j <= N - K; j++) {
        ans = max(ans, left_max[j] + right_max[j + 1]);
    }

    cout << ans << "\n";
    return 0;
}

الدرس السابع: عبقرية التعامل مع القيود ⚠️

كيف تمكن كودنا من اختراق قيود الـ Time Limit بسلام؟

1. قيد الوقت (N ≤ 500)

الطريقة التقليدية تأخذ O(N6). يعني $1.5 \times 10^{16}$ عملية (بطيء جداً).
أما الكود الخاص بنا (باستخدام Prefix Sums وتقسيم المزرعة)، فهو يمر على المزرعة بـ Loops متداخلة مرتين فقط لبناء المصفوفات. التعقيد الزمني هو O(N2).
العمليات هي $500 \times 500 = 250,000$ عملية. الكمبيوتر ينهيها في 0.005 ثانية! متجاوزاً قيد الـ 1 ثانية باكتساح.

2. حساب المجموع في لحظة

المشكلة الكبرى كانت في حساب مجموع مربع $K \times K$ في كل مرة، وهذا كان يستهلك $O(K^2)$. باستخدام مصفوفة المجاميع التراكمية pref، تقلص الوقت من مرور ممل على مئات الخلايا إلى 4 عمليات حسابية بسيطة (a - b - c + d). هذا سرّع الأداء بشكل خرافي.

الفخ الأول

💣 فخ السعة وتدمير الأرقام (Integer Overflow)

لاحظ القيود: قيمة المحصول الواحد تصل إلى $10^9$. إذا كان المربع بحجم $500 \times 500$ وكل القيم موجبة، فالمجموع سيصل إلى $250,000 \times 10^9 = 2.5 \times 10^{14}$. المتغير العادي int ينكسر وينفجر عند $2 \times 10^9$! إذا استخدمت int ستحصل على أرقام سالبة غريبة وتفشل في التقييم.

int sum = 0; // خطأ كارثي
long long sum = 0; // الحل الصحيح (سعة تصل لـ 10^18)
الفخ الثاني

☠️ التهيئة الخاطئة للقيم السالبة

قد تكون المزرعة كلها محاصيل فاسدة (قيم سالبة)، والجواب الأفضل هو أقل خسارة ممكنة (مثلاً -50). إذا قمت بتعريف المتغيرات للبحث عن الـ Max بوضع قيمتها الابتدائية 0، فإن الكود سيختار الصفر ويعتبره الجواب الأكبر، وهذا خطأ!

long long ans = 0; // خطأ: الجواب قد يكون سالب
long long ans = -1e18; // الصح: نبدأ من سالب ما لا نهاية
الفخ الثالث

🕸️ الفوضى في المؤشرات (0-Based vs 1-Based)

في خوارزمية Prefix Sums، استخدام مؤشرات تبدأ من 0 يضطرك لكتابة شروط if مزعجة لمنع الكود من الخروج عن حدود المصفوفة السالبة (Out of Bounds). أفضل طريقة لتجنب الأخطاء والمخالفات المزعجة هي جعل حجم المصفوفة N+1 وبدء الحلقات من 1، ليكون الصفر مجرد هامش آمن.
