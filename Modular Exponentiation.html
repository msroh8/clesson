الدرس الأول: تحليل المسألة الرياضية

فهم المطلوب والقواعد الرياضية التي تحكم باقي القسمة.

المطلوب في المسألة 🎯

لديك رقمان صحيحان n و m. المطلوب هو طباعة ناتج المعادلة التالية:

m % (2^n)

قاعدة رياضية هامة جداً للمسابقات:

بالنسبة لأي رقمين x و y، إذا كان المقسوم عليه أكبر من المقسوم (y > x)، فإن ناتج باقي القسمة هو الرقم الأول نفسه.

If (y > x) ⇒ (x % y == x)
  • مثال 1: 5 % 10 = 5 (لأن الـ 10 لا يمكن أخذها من الـ 5).
  • مثال 2: 14 % 100 = 14.

الدرس الثاني: فخ تجاوز السعة (Integer Overflow)

لماذا الحل المباشر يؤدي إلى فشل البرنامج (Wrong Answer)؟

⚠️ لماذا لا نستخدم دالة الأس المباشرة؟

التفكير المبتدئ هو حساب 2^n أولاً، ثم إيجاد باقي القسمة. ولكن في المسابقات البرمجية، قيمة n قد تكون ضخمة جداً (مثلاً n = 10^8).

حدود الـ int في C++

المتغير من نوع int لا يمكنه تخزين رقم يتجاوز 2^31 - 1. أي رقم أكبر من ذلك سيسبب خطأ تقنياً يسمى (Overflow)، وستتحول القيمة إلى أرقام سالبة عشوائية!

حدود الـ long long

حتى المتغير الأكبر long long لا يستطيع استيعاب أرقام مثل 2^{100000}. حجم هذا الرقم يفوق عدد ذرات الكون، والكمبيوتر سيفشل في تخزينه.

الدرس الثالث: الخوارزمية المحسنة (الإنهاء المبكر)

كيف نستغل القاعدة الرياضية للنجاة من الفخ؟

استراتيجية الحل المتقدمة 💡

بدلاً من حساب 2^n دفعة واحدة، سنقوم بمضاعفة المتغير خطوة بخطوة داخل حلقة تكرارية (Loop). وأثناء هذه العملية، سنقوم بوضع نقطة تفتيش (Checkpoint) لحماية البرنامج.

آلية الإنهاء المبكر (Early Termination):

  • نبدأ بمتغير power = 1 ونضربه في 2 مع كل دورة.
  • في كل دورة نسأل: هل أصبحت قيمة power > m ؟
  • إذا كانت الإجابة نعم، نتذكر القاعدة التي درسناها: أي رقم يتم قسمته على رقم أكبر منه، الباقي هو الرقم نفسه!
  • لذا، نطبع m فوراً، ونكسر الحلقة باستخدام أمر break لنوفر وقت الكمبيوتر ونحميه من الـ Overflow.

الدرس الرابع: تفكيك الكود الأساسي

ترجمة الاستراتيجية إلى كود C++ نظيف وفعّال.

اضغط على علامة التعجب ! بجوار الكود لتفهم وظيفة كل كتلة برمجية.

  • int power = 1;
    نهيئ المتغير التراكمي بقيمة 1 كبداية للحسابات (لأن 2 أس 0 يساوي 1).
  • power *= 2;
    داخل الحلقة، نضرب المتغير في 2 في كل دورة لتوليد القوى المتتالية (2، 4، 8، 16...).
  • if (power > m)
    الشرط الحارس! إذا تجاوزت قوة الـ 2 قيمة m، نطبع m مباشرة لأن m % (رقم أكبر منه) = m، ونخرج من الحلقة بـ break.
  • if (power <= m)
    إذا انتهت الحلقة كاملة ولم يتجاوز الرقم قيمة m، فهذا يعني أن الرقم صغير وآمن، ويمكننا إجراء عملية باقي القسمة المباشرة m % power بأمان.
#include <iostream>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;
    
    int power = 1; // المتغير التراكمي
    
    // حساب القوى بشكل تدريجي
    for (int i = 0; i < n; i++) {
        power *= 2;
        
        // شرط الإنهاء المبكر لمنع الـ Overflow
        if (power > m) {
            cout << m << endl;
            break; 
        }
    }
    
    // في حال كانت الدورة مكتملة بدون تجاوز m
    if (power <= m) {
        cout << m % power << endl;
    }
}

int main() {
  int n = 4, m = 10;
int power = 1;
for (int i = 0; i < n; i++) {
power *= 2;
if (power > m) {
cout << m << endl;
break;
}
}
if (power <= m) {
cout << m % power << endl;
}
}

التعقيد الزمني (Time Complexity)

بفضل تقنية الإنهاء المبكر (Early Exit)، الدورة لن تتجاوز أبداً O(log m). أقصى قيمة لمتغير int تتطلب حوالي 30 دورة فقط حتى يتجاوزها المتراكم. لذا، البرنامج سريع جداً ويعتبر حلاً نموذجياً للمسابقات!
